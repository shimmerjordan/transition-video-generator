# 转场视频生成器(transition-video-generator)— 实施计划

> **定位**:一个通用的 AI 转场视频生成管线(卡点 + 背景替换 + 换装 + 重打光 + 真实感合成),
> 做成**围绕付费AI产品 / 本地工具的可配置编排器**。下文以**首个用例**——双人舞卡点换背景/换装/换光影——驱动设计。

---

## 架构 v2(可配置编排器 + 标准件 + Web 前端)

根据真实素材与需求,项目定型为**导演式编排器**,三条支柱:

1. **标准件契约**(`docs/ARTIFACTS.md`):每步读写固定格式/路径的标准件(帧序列 PNG + JSON sidecar + manifest),保证可替换、可校验。
2. **Provider 抽象(每步 本地 or 现成产品,统一 I/O)**:`config.providers.<step>` 选 `local` 或 `product:<name>`。
   - `local`:项目内实现,直接产出标准件。
   - `product:*`:走 **handoff**——导出标准输入 → 你在付费产品里处理 → 结果放回 `ingest/` → 归一化成标准输出。产品有 API 时可全自动。
3. **Web 配置前端**(`src/webui.py`,FastAPI):编辑配置、为每步选 provider、按段配置(背景子片段/换装/地面)、触发运行、浏览各步产物。

### 真实素材带来的新增环节(预处理子管线)

真实 `dance.mp4`/`bg1`/`bg2` 不是干净素材,需在主合成前预处理(均为可配置、可选 provider 的步骤):

- **真抠像(matte)**:dance 是真实复杂场景,中值法不可用 → SAM2(本地)或付费 rotoscope;**按人身份**输出 per-person alpha(换装需要)。
- **背景片段选取(clips)**:每个背景视频**可配置取哪些时间段**(可多段),分配给不同节拍段。
- **去元素(cleanup)**:去 bilibili 水印/字幕(静态水印可本地 `cv2.inpaint` + 固定 mask)、去路人(视频修复,付费或本地 video-inpaint)。
- **地面/场景(ground)**:**按段策略可配**——有地面直接用 / 生成式补地面(outpaint)/ 虚拟地面平面 + 接触阴影。
- **稳定与运镜匹配**:沿用 S2 锁定域策略。

### 换装可配置(按人按段)

`config.segments[*].garments` 支持**按人身份指定**该段每个人换哪张参考图(或不换),与背景切换同步。人物身份来自 matte 的 per-person 轨迹(p0/p1…)。

## Context(为什么做、要达成什么)

首个用例:你有一段**固定机位(三脚架)**拍摄的双人舞视频,想要:

- 按一段音乐的**节拍卡点**,在节拍点切换背景;
- 背景从**你提供的其他视频**里截取(这些素材运镜与原片不一致);
- 两人的**衣服**随之切换(换成你给的参考图款式,原衣服为紧身);
- 光影随背景切换;
- **最高目标:整体真实,像"当时就在那个地方实拍的"**。

核心判断:这是一个**合成级真实感(compositing)**任务,不是简单换背景。真实度的命门依次是 ①**前景与背景共享同一摄像机运动**(运镜匹配)②光影一致(relighting)③边缘融合与接触阴影 ④镜头质感统一(颗粒/景深/运动模糊)。

工作区现状:`e:\data\workspace` 对视频处理是**全新**的(无任何视频/AI 依赖、无素材、无 GPU 配置),需从零搭建一个独立 Python 项目。GPU 为中低端 N 卡(8–12GB),分段短片处理可适配。

### 运镜匹配(通用化核心,源片可能有视角移动)

源视频**未必固定机位**——可能有摇/俯仰/滚转/变焦甚至推轨。前景一旦有运镜,替换的背景必须**共享同一套摄像机运动**,否则立刻穿帮。通用策略:

> **「稳定化 → 锁定域合成 → 把源片运镜重新贴回成片」**
> 1. 用特征点跟踪(排除主体区域)估计源片每帧摄像机运动 `T_t`(homography/相似变换);
> 2. 把源片**反稳定到锁定域**——抠像/换装/打光/合成全部在"静止画面"里做(最易做真);
> 3. 背景素材同样稳定成 locked plate;
> 4. 合成完后,把 `T_t` **整体重新施加到成片**(前景+背景一起)→ 二者天然同运镜。需 **overscan(留黑边余量)**防止边缘露空。

- 三种相机模式(config 选 `auto`):`locked`(固定机位,跳过运镜处理)/ `motion_2d`(上面的 2D 反贴策略,覆盖绝大多数旋转+变焦运镜)/ `motion_3d`(高级:仅当**大幅推轨产生强视差**时,用 COLMAP / 单目 SLAM 解算相机 + 深度平面重投影)。
- 默认 `motion_2d`,这是通用性与算力的最佳平衡。

利好简化点:
- 原衣服紧身 → 姿态/深度约束稳;注意新衣服更宽松时重绘区需外扩补绘。
- 卡点是**硬切换装**(段内衣服不变)→ 换装按"节拍段落"分块,规避了长镜头连续换装的时序闪烁难题。

## 技术路线(分段流水线)

```
music ──librosa──► beats.json(节拍时间戳)
                        │ 按节拍把源片切成 N 段
source.mp4 ──跟踪──► 每帧运镜 T_t  ──反稳定──► 锁定域源片
           └──SAM2──► 每帧主体 alpha/mask(多主体,带时序追踪)
backgrounds/ ──稳定化+裁切──► 每段 locked plate(去掉背景自身运镜)
                        │  (以下全部在锁定域进行)
        ┌── 每段独立处理(段内衣服/背景固定)──┐
        │  换装(ControlNet+参考图,段内一致)   │
        │  relighting(IC-Light,按背景光照打光) │ ★真实度核心
        │  合成(light wrap / 接触阴影 / 颗粒 / 景深) │
        └────────────────────────────────────┘
                        │ 把 T_t 重新贴回(overscan)→ 前景背景同运镜
            按 beats 转场拼接 + 配乐 ──► 成片
```

## 各步骤实现要点与工具

### S1 卡点 `s1_beats.py`
- `librosa.beat.beat_track` 得到 BPM 与节拍时间戳;支持手动微调/降采样(不一定每拍都切)。
- 输出 `beats.json`(秒 + 帧号),作为后续所有切段依据。

### S2 运镜跟踪与反稳定 `s2_camera.py`(★通用化核心,源片可能有视角移动)
- **相机模式自动判定**:用全帧特征点位移幅度判定 `locked` / `motion_2d` / `motion_3d`(也可在 config 强制)。
- `motion_2d`(默认):特征点跟踪求每帧相对参考帧的 **homography/相似变换 `T_t`**;**用 S3 的主体 mask 把舞者区域排除**在跟踪之外(否则人动会污染相机解)。把源片按 `T_t⁻¹` **反稳定到锁定域**,后续 S3–S7 全在锁定域做。`T_t` 存盘供 S8 贴回。
- `motion_3d`(高级,仅强视差时):COLMAP / 单目 SLAM(如 DROID-SLAM)解算相机内外参 + 单目深度,背景按深度平面重投影,产生真实视差。算力高,作为退路。
- `locked`:跳过本步,直接进 S3。

### S3 抠像 `s3_segment.py`
- **SAM 2**(视频分割,带时序追踪)抠多主体,遮挡靠时序传播比逐帧稳;主体数量由 config 定。
- 在**锁定域**上抠像(画面已静止,matte 更稳);并把 mask 回传给 S2 用于排除主体区域跟踪(S2/S3 可迭代一次)。
- 8–12GB 显存:按段送入(短片),必要时降分辨率推理再上采样 alpha。
- 输出每帧 RGBA / alpha matte;边缘羽化 + 轻微腐蚀防白边。

### S4 背景锁定平面 `s4_plates.py`
- 对每段背景素材做**视频稳定化**(OpenCV 特征点 + 全局运动估计或现成 stabilizer),消除其自身运镜 → 得到 locked plate,与锁定域前景对齐。
- 裁切/缩放到目标分辨率构图;必要时补帧/循环覆盖段长;**预留 overscan**(S8 贴回 `T_t` 时背景会被移动,需多出画面余量防露空)。
- 从平面**估计主光方向与色温**(亮度梯度/高光分析),传给 S6。

### S5 换装 `s5_garment.py`(瓶颈环节,可选)
- `enable_garment_swap=false` 时跳过。思路:**段内固定款式的视频重绘**。SD1.5 + **ControlNet(DWPose 姿态 + Depth)** 锁动作与身体结构,服装参考图作外观引导(IP-Adapter / 参考图条件)。
- 紧身原衣 → 姿态/深度准;**新衣更宽松时把换装 mask 外扩(dilate),露出的旧背景在合成层由 plate 覆盖**,避免补绘穿帮。
- 时序一致:段短 + 固定 seed + 时序约束(AnimateDiff 或逐帧 + 光流平滑);多主体分别处理再合并。
- **风险与退路**:最易翻车。退路 ① 段切更短;② 崩帧在 DaVinci 手动修补;③ 这一步单独上云。

### S6 重打光 `s6_relight.py`(★真实度核心,可选)
- `enable_relight=false` 时跳过。**IC-Light**:以 S4 估计的背景光照(方向/色温/强度)为条件,对(换装后)主体重新打光,使人景光照统一。
- "假/真"的分水岭——务必用背景的真实光照参数驱动,而非凭感觉。

### S7 合成 `s7_composite.py`(锁定域,让它"像实拍")
- **Light wrap**:让背景边缘光"溢"到主体轮廓,消除剪贴感。
- **接触阴影 / 投影**:在主体脚下/接触面补软阴影(按主光方向投射),否则像贴上去。
- **镜头质感统一**:对主体层与背景层施加统一的**胶片颗粒、运动模糊量、景深(背景虚化)**;统一色彩(color match / LUT)。
- 边缘羽化 + alpha 微调。建议**精修在 DaVinci Resolve 或 Nuke 里手动收尾**,自动化做大头、手动保真。

### S8 贴回运镜 + 拼接出片 `s8_restore_assemble.py`
- 把每段合成结果按 `T_t` **整体重新施加源片运镜**(前景背景一起 warp,`locked` 模式跳过),裁掉 overscan 余量 → 前景背景天然同运镜。
- 按 `beats.json` 在节拍点按 `transition`(硬切/交叉/遮罩)拼接,封装音乐,ffmpeg 导出。

## 验证(先 POC,再铺开)

**强制要求:先跑通一个节拍段(3–5 秒)的全链路,再批量。**

1. `pipeline.py --segment N` 单段模式:取一个**有运镜 + 有主体遮挡**的代表段,跑 S1→S8。
2. 重点验收三件事:
   - **运镜匹配**:贴回 `T_t` 后,前景与背景是否同步运动、无"背景滑动/前景漂移"穿帮;
   - **换装**在遮挡帧是否稳定、款式是否还原;
   - **打光+合成后人景是否融**(光向一致、有接触阴影、颗粒/景深统一)。
3. 主观对比:把成片单帧/动态与"真在该场景实拍"的参考感觉对照,检查穿帮点(运镜、边缘、光向、阴影、色温)。
4. POC 通过 → 固化 `config.yaml` 参数,批量跑全部段 → S8 拼接 → 整体审片。

验收标准:动起来前景背景**同运镜不分离**、无换装闪烁/抖动;静帧经得起细看(无白边/无错误光向/有接触阴影)。

## 主要风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| 源片运镜跟踪不准 → 前景背景分离穿帮 | 高(直接关乎真实度) | 主体 mask 排除跟踪;强视差段退 `motion_3d`;实在不行手动稳定 |
| 强视差(大幅推轨)2D 反贴不够 | 中 | 切到 `motion_3d`(COLMAP/SLAM+深度重投影)或回避此类镜头 |
| 视频精确换装在多主体遮挡时崩帧 | 高 | 切更短段 / 手动修补 / 该步上云 |
| 中低端显存跑大模型 | 中 | 分段短片 + offload + 降分辨率推理 |
| 背景稳定化/贴回运镜后边缘黑边 | 中 | 预留 overscan、适度裁切放大、补帧、循环平面 |
| 光照估计不准导致人景不融 | 高(直接关乎真实度) | 手动指定主光方向/色温覆盖自动值 |
| Python 3.13 生态兼容 | 低 | 改用 3.10/3.11 venv |

## 落地分工(混合)

- **自动化(代码,本地为主)**:S1 卡点、S2 运镜跟踪/反稳定、S3 抠像、S4 稳定化、S5 换装、S6 打光、S8 贴回运镜/拼接 —— 做"量"。
- **手动精修(DaVinci/Nuke)**:S7 合成收尾、崩帧修补、最终调色,以及个别难解镜头的手动稳定/match-move —— 做"质",保真实感。
